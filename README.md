#科技脈動 Blog - Cloudflare Pages (Frontend)這個儲存庫包含一個單一的 index.html 檔案，它是一個完全自給自足的前端應用程式，用於 Cloudflare Pages 的靜態部署。重要說明：本應用程式是一個前端介面，需要您的 Cloudflare Worker (D1/KV 後端邏輯) 協同運作才能運作。部署到 Cloudflare Pages 的步驟由於您只有一個 index.html 檔案，部署過程非常簡單，但需要設定 Pages Functions (Worker) 路由來指向您的後端 API。1. 建立 Pages 專案登入您的 Cloudflare 儀表板。前往 Pages 服務，點擊 建立專案 (Create a project)。選擇 連線到 Git (Connect to Git) 並選擇此 GitHub 儲存庫。輸入專案名稱 (例如：tech-pulse-blog-frontend)。在 建構設定 (Build settings) 區塊：建構指令 (Build command): 留空建構輸出目錄 (Build output directory): 留空或輸入 ./ (因為檔案直接位於根目錄)點擊 儲存並部署 (Save and Deploy)。此步驟會部署您的前端。2. 設定 Pages Functions (API 路由)Pages Functions 允許 Pages 專案將特定路徑的請求代理或轉發到一個 Worker。在您的 Pages 專案概覽頁面，點擊 設定 (Settings)。導航到 Functions 選項卡。在 Functions 區塊中，您需要告訴 Pages 將所有 /api/* 請求轉發到您的後端 Worker。點擊 新增規則 (Add Rule) 或找到 路由 (Routes) 設定。路徑 (Route): api/*處理程式 (Handler/Service): 選擇您的 已部署 Worker 服務名稱。 (例如：my-blog-worker)一旦設定完成：當瀏覽器請求 your-page-domain.com/api/posts 時，Cloudflare Pages 會自動將該請求代理到您的後端 Worker 服務，實現資料的 CRUD 操作。應用程式技術棧Frontend: HTML5, JavaScript (Vanilla JS), Tailwind CSS (CDN)Backend (Required): Cloudflare Worker (處理 D1 查詢, KV 儲存)Database: Cloudflare D1 (儲存文章和評論)Storage: Cloudflare KV (儲存用戶憑證)登入測試：使用用戶名：admin，密碼：kai1020622 測試管理員功能。
